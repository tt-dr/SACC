# SACC.pen 对照审计

## 原始问题

在本次改造之前，项目存在几类核心偏差：

1. `sacc-home` 在运行时直接读取 `SACC.pen`，把设计稿当页面数据源。
2. 大量页面交互依赖节点名称、文本内容和祖先上下文猜测跳转逻辑。
3. 首页和加入我们页是手写实现，其余栏目页主要依赖 Pen 渲染，形成双数据源。
4. 公共导航、页脚、联系方式和栏目内容在多个页面重复硬编码。
5. 招新页存在占位群号、占位二维码等不适合正式官网上线的信息。

## 本次修正

本次改造后，官网前台的策略变为：

- `SACC.pen`
  保留为设计校验和视觉参考源，不再承担线上运行时路由职责。
- `sacc-home`
  改为显式路由 + React 页面组件 + 统一内容模型。
- `server`
  提供后端内容和站点元信息的 API 入口，前台可直接对接。

### 与设计稿保持一致的部分

- 保留了暖橙 + 深蓝 + 浅米色的整体气质。
- 首页仍然具备 Hero、统计、特色、活动、动态、成员反馈、合作伙伴、资源、CTA、Footer 等完整节奏。
- 关于、活动、项目、团队、相册、FAQ、加入我们等栏目页保留了设计稿的内容分区逻辑。

### 与设计稿主动做的工程化修正

- 把纯视觉模块改成了结构化内容区，方便后续接 CMS/API。
- 把详情页从“依赖文字按钮猜路由”改成了真实 URL 映射。
- 把团队页 tab 的样式推断逻辑改成了显式的路由分支。
- 把官网基础信息统一收敛到 `siteContent` 和后端 bootstrap 接口。

## 当前结论

官网已经从“设计稿播放器”升级成“可维护的官网应用”，后续设计改版时可以继续参考 `SACC.pen`，但不需要再把它当线上运行时单点依赖。
